# Digital Diary Backend - Developer Guide

This guide outlines the commands and workflows for developing, testing, and deploying the Digital Diary Python Backend.

## 1. Prerequisites
- **Docker Desktop** (Mac/Windows) or **Docker Engine** (Linux)
- **Git**

## 2. Local Development (Mac/Linux)

### Start the Application
1. **Start Database**:
   ```bash
   docker compose up -d db
   ```
2. **Run Migrations** (If you have pending schema changes):
   ```bash
   docker compose run --rm app alembic upgrade head
   ```
3. **Start API**:
   ```bash
   docker compose up
   # Add --build if you changed requirements.txt or Dockerfile
   docker compose up --build
   ```
- API URL: `http://localhost:8000`
- API Docs: `http://localhost:8000/docs`
- DB Port: `5432`

### Database Migrations (Schema Changes)
We use **Alembic** for migrations. The workflow is:

1. **Modify Models**: Edit files in `app/models/`.
2. **Generate Migration Script** (Auto-detect changes):
   ```bash
   docker compose run --rm app alembic revision --autogenerate -m "describe_your_change"
   ```
3. **Review Script**: Check the new file in `alembic/versions/` to ensure it looks correct.
4. **Apply to Local DB**:
   ```bash
   docker compose run --rm app alembic upgrade head
   ```
5. **Commit**: `git add alembic/versions/*.py` and commit.

### Useful Commands
- **Check Logs**:
  ```bash
  docker compose logs -f app
  ```
- **Access DB Shell**:
  ```bash
  docker compose exec db psql -U admin -d digital_diary
  ```
- **Reset Database** (Caution: Deletes all data):
  ```bash
  docker compose down -v
  docker compose up --build
  ```

## 3. Production Deployment (Ubuntu Server)

**CRITICAL RULE**: Never run `alembic revision` on production. Only apply existing migrations.

### Deployment Steps
1. **Pull Latest Code**:
   ```bash
   cd digital-diary-backend
   git pull origin main
   ```

2. **Apply Database Migrations**:
   This updates the production DB schema to match the code.
   ```bash
   docker compose run --rm app alembic upgrade head
   ```

3. **Rebuild and Restart Containers**:
   ```bash
   docker compose up -d --build
   ```

### Troubleshooting Production
- **Check Status**: `docker compose ps`
- **View Logs**: `docker compose logs -f --tail=100`

## 4. 🤖 AI & Embeddings

### Switching Models
**CRITICAL**: Embeddings from different models (e.g., `gemini-embedding-001` vs `text-embedding-3-small`) are **incompatible**.
-   If you verify a new model, you **cannot** compare its output to vectors generated by the old model.
-   **Strategy**: If you change the model in `app/core/ai_service.py`, you MUST:
    1.  Update the `Vector(dimension)` in `app/models/contact.py` (if the dimension changes, e.g., 768 -> 1536).
    2.  Write a script to **re-generate embeddings** for ALL existing contacts using the new model.
    3.  This is possible because we store the source text (name, title, org) in the database.
